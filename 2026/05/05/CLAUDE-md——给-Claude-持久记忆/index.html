CLAUDE.md——给 Claude 持久记忆

1、什么是 CLAUDE.md？

CLAUDE.md 是 Claude Code 每次启动时自动读取的配置文件，用于持久化项目规范，免去每次会话重复说明的成本。

文件位置 作用范围 适合存放的内容
项目根目录/CLAUDE.md 当前项目，提交到 Git 技术栈、编码规范、常用命令、架构说明（团队共享）
.claude/CLAUDE.md 当前项目（更整洁） 同上，推荐结构
CLAUDE.local.md 当前项目，不提交 个人本地配置，不共享给团队
~/.claude/CLAUDE.md 全局，所有项目 个人习惯、语言偏好、通用工作流

4、写作原则

  • 写规则而非故事：「使用命名导出」比「我们团队历史上选择命名导出，因为……」更有效
  • 聚焦 Claude 最容易犯的错：不需要面面俱到，把精力花在高风险点上
  • 重要规则加强调：用 IMPORTANT: 或 YOU MUST 前缀，确保 Claude 不会忽略
  • 禁止比建议更有效：明确红线比模糊偏好更能约束行为
  • 定期维护：技术栈或规范变化时及时更新，避免过时指令误导
